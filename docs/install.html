Install Guide

Get GuardRailDB running in under two minutes.

Prerequisites

  • Node.js v18 or later
  • A running PostgreSQL instance (local or cloud, e.g. Supabase)
  • Your database connection string (postgresql://user:pass@host:5432/dbname)

Quick Start

Run with a single npx command — no installation required:

npx db-guardrail-mcp --connectionString=postgresql://user:pass@localhost:5432/mydb

Step by Step

1. Get an API Key

Sign up at guardraildb.dev/signup to receive your API key. In Phase 1 the key is accepted but not yet enforced — providing it now means your config won't change when validation goes live.

2. Run the Server

npx db-guardrail-mcp \
  --connectionString=postgresql://user:pass@localhost:5432/mydb \
  --apiKey=your_key_here

3. Confirm It's Running

You should see a version banner and the server will wait for an MCP client to connect. No news is good news — if there are no connection errors, you're ready.

Troubleshooting

ProblemSolution
Cannot connect to databaseVerify your connection string — check host, port, credentials, and that your IP is allowed (for cloud databases like Supabase, enable IPv4 in project settings).
Invalid API keyGenerate a new key from your account page. In Phase 1 you can omit --apiKey.
Node.js version errorEnsure you're on Node.js v18+. Run node --version to check.
