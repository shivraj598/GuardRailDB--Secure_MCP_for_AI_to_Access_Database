GuardRailDB

A secure, schema-aware MCP gateway connecting LLMs and AI agents to Postgres databases through a policy-controlled gateway.

Under production — deploying after security audit.
Get Started GitHub

Why GuardRailDB?

Every team wiring an AI agent to a database runs into the same two failure modes: dumping the whole schema into the prompt (burns tokens, slows the model) or letting the model write its own SQL (one bad prompt injection away from a dropped table). Most solutions fix one problem and ignore the other.

GuardRailDB is a proxy layer that sits between your AI client and your database and refuses to accept either tradeoff. It speaks the Model Context Protocol natively, so any MCP-compatible client gets structured, validated, pre-filtered access.

Core Features

Schema Compacting

Queries information_schema and compresses it into dense, LLM-readable Markdown — cutting token usage for schema context.

Strict Input Validation

Every tool call is parsed and validated against a Zod schema before becoming a safe, structured query. No SQL injection surface.

Hard Pagination

Record requests are capped at 50 rows server-side, preventing runaway or malicious queries from flooding memory.

Sandboxed Simulation

Mutating queries run inside a transaction that is immediately rolled back — zero risk to persistent data.

Architecture

The server runs over MCP's stdio transport — spawned as a local child process by the calling client. Your connection string, schema, queries, and results never leave your machine. The only hosted infrastructure is a lightweight key-validation endpoint and this documentation site.
